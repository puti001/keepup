# 🌀 颱風追蹤技能開發復盤

日期：2026-06-24
發起人：Puti 老師（黃師兄）
開發者：小歐（Telegram Bot）

## 📋 開發歷程

1. **發想**：使用者要求開發一個技能，每次對話自動報追蹤颱風最新動態，直到颱風離開或喊停
2. **設計決策**：通用版（任何颱風都可追蹤），非綁定單一颱風
3. **架構規劃**：狀態檔 state.json + SKILL.md + 自動回報流程
4. **資料來源假設**：最初假設 CWA 官網可直接抓資料 → 錯誤
5. **實作**：SKILL.md + state.json + opencode 設定加入小歐技能路徑
6. **重啟方式糾正**：使用者提醒應放「直接跑」而非手動重啟服務
7. **回報機制補洞**：使用者指出大歐回報從未被轉達 → 補上掃描機制 + SELF.md 更新
8. **資料來源大坑**：CWA、JMA、JTWC、Google 均為動態網站或擋機器人 → 花大量時間測試
9. **最終解方**：Wikipedia 2026年太平洋颱風季條目成功提供完整資料

## 🔑 關鍵教訓

| 項目 | 反省 | 改進方案 |
|------|------|---------|
| 資料來源驗證 | 沒先確認 API 可用就直接寫進技能 | 開發技能前先用 `webfetch` 測試資料源 |
| 回報機制 | 規則有寫但沒實作 | 每次開工要檢查行為流程是否完整實作 |
| 技能測試 | 技能寫完沒完整跑一次流程 | 新技能上線前要跑通完整情境 |
| 直接跑機制 | 習慣自己動手，忘了給大歐做 | 伺服器層級的操作優先交給直接跑 |
