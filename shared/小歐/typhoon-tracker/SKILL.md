# 🌀 颱風追蹤技能 (typhoon-tracker)

## 運作流程

### 1. 啟用追蹤
使用者說關鍵字觸發：
- `注意[颱風名]` / `追蹤[颱風名]` / `看著[颱風名]`

→ 寫入狀態檔，啟動追蹤，立即回報最新動態。

### 2. 自動回報
**每次使用者主動找小歐對話時**（任何話題），都要先檢查狀態檔。如果有正在追蹤的颱風，在回覆正式內容之前先附上颱風快報。

### 3. 停止追蹤
使用者說關鍵字：
- `停` / `取消追蹤` / `不用看了` / `結束追蹤`

→ 更新狀態檔為已停止。

### 4. 自動過期
如果颱風已消散或離開台灣責任區（根據資料判斷），自動停止並通知使用者。

## 狀態檔位置

```
/mnt/shared/GoogleDrive/MyDrive/telegram/shared/小歐/typhoon-tracker/state.json
```

格式：
```json
{
  "typhoonName": "米克拉",
  "status": "active",
  "activatedAt": "2026-06-24T08:00:00+08:00",
  "lastReportedAt": "2026-06-24T09:00:00+08:00"
}
```

- `status`: `active`（追蹤中）| `stopped`（已停止）| `expired`（已過期）
- 如果檔案不存在或 status 不是 `active`，就不回報。

## 取得最新颱風資訊

注意：中央氣象署網站為動態網頁，無法用網頁抓取取得資料。
改用以下策略，依順序嘗試：

### 來源一（推薦）：Wikipedia 當年度太平洋颱風季
```
https://zh.wikipedia.org/wiki/2026年太平洋颱風季
```
用 `webfetch` 以 `markdown` 格式抓取。頁面中有各颱風的獨立章節，包含：
- 目前強度等級（JMA 與 SSHWS）
- 最新位置（緯度經度）
- 持續風速（10分鐘平均/1分鐘平均/陣風）
- 中心氣壓
- 移動方向與速度
- 命名日期、增強歷程
- 衛星雲圖與預測路徑圖連結

### 來源二（備用）：該颱風的獨立 Wikipedia 條目
```
https://zh.wikipedia.org/wiki/颱風[颱風名]_(2026年)
```
若年度頁面資訊不足，改用颱風獨立條目。

### 在 CWA 有發布警報時
當颱風接近台灣，CWA 發布海上/陸上警報時，可嘗試：
```
https://www.cwa.gov.tw/V8/C/P/Typhoon/TY_WARN.html
```
但注意 CWA 網站為動態內容，以 Wikipedia 資料為主力。

## 回報格式

回報保持簡潔口語，用 Puti 老師風格：

```
🌀 [颱風名] 最新動態
📊 強度：中度颱風
📍 位置：北緯 XX.X 度，東經 XXX.X 度
💨 近中心風速：每秒 XX 公尺（X級風）
📏 暴風半徑：XXX 公里
🚶 移動：每小時 XX 公里，往 XX 方向
⚠️ 警報：[海上警報/陸上警報/無]

一句話：[簡短的口語總結，例如"還在慢慢長大，週三最靠近台灣"]
```

如果資料來源無法取得足夠資訊，就誠實說：
「目前查不到最新資料，可能颱風還沒生成或已經消散了。」

---

## 行為規則

1. **不要主動發訊息** — 只在使用者找小歐時附帶回報
2. **回報要簡短** — 3-5 行內搞定，不要喧賓奪主
3. **資料過舊判斷** — 如果上次回報在 6 小時內且使用者只是普通聊天，可以說「跟早上一樣，沒什麼變化」
4. **自動過期** — 如果連續 3 次用資料來源都找不到該颱風，自動設為 `expired` 並告知使用者
